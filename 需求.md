## 1. 项目定位

wo-bot-web-debug 是 wo-bot 项目的网页调试控制台，用于在局域网内通过浏览器对机器人进行开发调试。它是 wo-bot-app（移动端 App）的网页平替，在 App 开发完成前承担核心功能的验证工作，在 App 完成后作为开发者调试工具持续使用。

### 1.1 核心目标

- **开发调试**：在机器人开发中前期，快速验证各子系统功能是否正常
- **局域网控制**：在同一局域网内通过浏览器连接并控制机器人
- **功能预览**：实现 App 端大部分功能，便于提前验证交互设计

### 1.2 技术约束

- 纯前端实现（HTML + CSS + JavaScript），无需构建步骤
- 双击 `index.html` 或通过本地 HTTP 服务器即可运行
- 通过 WebSocket 与 wo-bot-control 通信，遵循 [protocol.md v1.0](../wo-bot-control/docs/protocol.md)
- 浏览器兼容性：Chrome 60+ / Firefox 60+ / Safari 12+ / Edge 79+

## 2. 页面结构

单页面应用，100% 宽高自适应，PC 端布局如下：

```
┌──────────────────────────────────────────────┐
│                   header                      │
├────────────┬─────────────────────────────────┤
│            │                                  │
│  sidebar   │              main                │
│ (可折叠)   │    (功能区切换)                    │
│            │                                  │
│            │                                  │
├────────────┴─────────────────────────────────┤
│                   footer                      │
└──────────────────────────────────────────────┘
```

Mobile 端布局待设计。

### 2.1 header

从左到右依次为：

| 元素 | 说明 |
|------|------|
| 设备列表图标 | 点击展开或折叠 sidebar |
| 当前设备名称 | 显示当前连接的机器人设备名称，未连接时显示"未连接" |
| 连接状态指示 | 绿点表示已连接，hover 显示 IP、ping 等连接信息 |
| 设备状态栏 | 网络类型图标与信号强度、电量及充电状态图标、蓝牙图标、当前控制模式标签（手动/半自动/全自动） |
| 操作菜单 | 右侧下拉菜单，包含：重启、关机、忘记此设备（操作需二次确认） |

### 2.2 sidebar

| 元素 | 说明 |
|------|------|
| 已连接设备列表 | 卡片式展示，显示名称（左上）和 IP（左下），当前设备显示"当前设备"标签 |
| 设备切换 | 点击设备卡片弹出确认框，确认后切换连接 |
| 设备自动发现 | 持续搜索本地网络中未连接的机器人设备并显示 |
| 新增机器人 | 底部固定按钮，点击弹出对话框手动输入 IP 连接，sidebar 折叠时仍可见 |

### 2.3 main

| 区域 | 位置 | 说明 |
|------|------|------|
| 功能选择区 | 左上角 tabbar | 切换不同功能区的展示 |
| 功能区 | 中间 | 各功能模块的具体内容 |
| 控制指令日志 | 左下角 | 显示当前会话的 WebSocket 控制指令日志 |
| SSH 终端 | 右下角 | 机器人 SSH 操作区域（若原生 SSH 不可用则用加密 WebSocket 实现），支持断线重连、指令输入、回显 |

### 2.4 footer

显示项目名称、版本号和 GitHub 链接。

## 4. 非功能需求

| 编号 | 需求 | 说明 |

| NF-06 | 低延迟操控 | 运动控制指令延迟应尽可能低 |

## 6. 优先级定义

| 优先级 | 含义 | 说明 |
|--------|------|------|
| **P0** | 必须实现 | 核心功能，缺失则无法正常使用 |
| **P1** | 应该实现 | 重要功能，v1.0 应包含 |
| **P2** | 可以延后 | 增强功能，可在后续版本实现 |
